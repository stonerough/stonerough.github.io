⚡ 30-Second OpenAthens Triage Checklist

Fast classification of authentication vs entitlement vs vendor UI failures

STEP 0 Frame the problem (2-3 seconds)

Ask yourself:

"Where is the user starting from?"
  • • Library link?
  • • Vendor site?
  • • Google / bookmark?

This immediately narrows the path.

STEP 1 Test the known-good path (10 seconds)

🔧
Pre-test: clean your URLs first

If the URL came from a user or a browser bookmark, it may have proxy layers wrapped around it. Strip those before testing, otherwise you're testing the wrapper, not the resource.

Use the OA URL Converter tool to strip proxy prefixes, construct a clean testable redirector URL, and identify the correct institutional entry point. Confirmed entry points for all Waikato subscribed databases are embedded in the tool and in the companion openathens_url_skill.md reference.

Open the OpenAthens redirector link in Libguides and/or discovery:

Libguides A-Z databases
libraryguides.waikato.ac.nz/az/databases
Primo discovery search
waikato.primo.exlibrisgroup.com/discovery/search
Typical OpenAthens redirector format
https://go.openathens.net/redirector/waikato.ac.nz?url=<target-url>
✅ Works

Authentication + entitlement are fine.

→ Move to Step 2
❌ Fails

Problem is NOT the vendor login UI.

  • OpenAthens config
  • IdP issue
  • Subscription entitlement
  • Vendor access setup
→ Real service impact

STEP 2 Test the vendor login path (10 seconds)

Go to the vendor site and try: "Sign in via your institution"

❌ Fails

Classic pattern:

✅ Redirector works
❌ Vendor login fails

→ Vendor WAYF / SP-initiated SAML issue
✅ Works

Both paths functioning.

→ Move to Step 3

STEP 3 Check entitlement (5-10 seconds)

If both login paths work: can you access actual content (PDF / article / book)?

❌ Blocked / paywalled
→ Entitlement / subscription issue
✅ Works
→ No system issue (user-specific / transient)
